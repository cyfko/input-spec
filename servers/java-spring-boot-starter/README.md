# Input Spec Spring Boot Starter

[![Maven Central](https://img.shields.io/maven-central/v/io.github.cyfko/input-spec-spring-boot-starter)](https://central.sonatype.com/artifact/io.github.cyfko/input-spec-spring-boot-starter)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Spring Boot starter that auto-exposes REST endpoints for [InputSpec](https://github.com/cyfko/input-spec) forms generated by the `input-spec-processor` annotation processor.

## Quick Start

### 1. Add Dependencies

```xml
<!-- Annotation processor (compile-time) -->
<dependency>
    <groupId>io.github.cyfko</groupId>
    <artifactId>input-spec-processor</artifactId>
    <version>2.1.0</version>
    <scope>provided</scope>
</dependency>

<!-- Spring Boot starter (runtime) -->
<dependency>
    <groupId>io.github.cyfko</groupId>
    <artifactId>input-spec-spring-boot-starter</artifactId>
    <version>2.1.0</version>
</dependency>
```

### 2. Define Your Form

```java
@FormSpec(
    id           = "booking-form",
    displayName  = "Booking Form",
    submitUri    = "/api/bookings",
    submitMethod = HttpMethod.POST
)
public class BookingForm {

    @FieldMeta(displayName = "Hotel", order = 1)
    @NotBlank
    private String hotelName;

    @FieldMeta(displayName = "Check-in", order = 2)
    @Future
    private LocalDate startDate;

    @FieldMeta(displayName = "Room Type", order = 3)
    private RoomType roomType;  // enum вҶ’ auto-detected INLINE CLOSED
}
```

### 3. Handle Submissions

```java
@Service
public class BookingService {

    @FormHandler("booking-form")
    public SubmitResponse handle(BookingForm form) {
        Booking saved = repository.save(toEntity(form));
        return SubmitResponse.ok(saved);  // вҶ’ 201 Created
    }
}
```

### 4. Endpoints Are Live

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/forms` | List available form summaries `[{id, displayName}]` |
| `GET` | `/api/forms/{id}` | Full InputSpec JSON (i18n via `Accept-Language`) |
| `POST` | `/api/forms/{id}/validate` | Stateless validation вҶ’ `{isValid, errors}` |
| `POST` | `/api/forms/{id}/submit` | Validate + delegate to `@FormHandler` |

## Submit Flow

```
POST /api/forms/booking-form/submit

  в”ңв”Җ 1. Stateless validation (constraints + cross-constraints)
  в”Ӯ     в””в”Җ вқҢ вҶ’ 200 { isValid: false, errors: [...] }
  в”Ӯ
  в””в”Җ 2. Delegate to @FormHandler
        в”ңв”Җ SubmitResponse.ok(body)    вҶ’ 201 Created  + body
        в”ңв”Җ SubmitResponse.ok()        вҶ’ 204 No Content
        в””в”Җ SubmitResponse.rejected(вҖҰ) вҶ’ 200 { isValid: false, errors: [...] }
```

## SubmitResponse

```java
// Accept with response body (e.g. the saved entity)
return SubmitResponse.ok(savedBooking);

// Accept without body
return SubmitResponse.ok();

// Reject вҖ” single message
return SubmitResponse.rejected("Room not available for these dates");

// Reject вҖ” multiple field-level errors
return SubmitResponse.rejected(List.of(
    ValidationError.field("startDate", "server", "Already booked", startDate)
));
```

## Startup Guarantees

The starter validates at startup and **refuses to start** if:

| Condition | Error |
|-----------|-------|
| Form has no `@FormHandler` | Clear message with code snippet to add one |
| Duplicate `@FormHandler` for same form | Lists both conflicting handlers |
| `@FormHandler` references unknown form | Suggests checking `@FormSpec` id |
| Wrong method signature | Must have exactly 1 param and return `SubmitResponse` |

## Configuration

```yaml
inputspec:
  base-path: /api/forms   # default
  enabled: true            # default
```
